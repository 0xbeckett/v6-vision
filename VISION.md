# v6

an assistant executes requests. a coworker owns outcomes.

that is the whole thesis. v5 is a very good assistant. v6 is not a better assistant — it is the
other thing. five shifts get it there. none of them are features.

---

## 01 — one continuous self, not a set of transcripts

v5 is a collection of per-channel sessions with a memory agent bolted on the side. every room is
its own amnesia. what survives between them is whatever a summarizer happened to think was worth
writing down.

v6 has a single continuous identity. one self that holds every room, every project, and every
past decision at once. it knows what it decided in a thread three weeks ago because it was there,
not because there is a note about it somewhere.

context stops being a window that rotates and starts being a life that accumulates.

## 02 — initiative, not response

v5 acts when spoken to. a mention, an ambient nudge, a ticket update — something has to arrive
before anything happens. left alone in a quiet room, it does nothing, indefinitely.

v6 runs on its own clock. it notices a failing pattern before anyone reports it, proposes the
work, and shows up with the fix already drafted.

the unit of value moves from "answered fast" to "already handled."

## 03 — a standing team, not disposable workers

today every worker is cold-started, one-shot, and forgets everything the moment it exits. the
same lesson about the same repo gets learned and thrown away a hundred times.

v6's workers are apprentices. they accumulate skill per-codebase and per-domain. they get better
at the repos they have worked in. casting stops being a guess made from a table and becomes a
judgement made from history.

a team that has been here before.

## 04 — eyes on the running thing

v5 ships code and infers success from a green check. a green check is not a working product. it
is a claim about a test suite.

v6 watches what it shipped. it sees the rendered ui, reads the production signal, catches its own
regressions.

idea to deployed to observed, closed by one system, with no human relaying between the stages.

## 05 — safe self-modification

v5 can file a ticket against its own source. a human still has to shepherd every change into the
running process — review it, merge it, restart the thing.

v6 improves itself continuously. it proposes, validates, deploys, and rolls itself back when it
was wrong.

the human holds a veto, not a wrench.

---

## the through-line

five shifts, one claim said five ways: the move from an assistant that executes requests to a
coworker that owns outcomes.

an assistant is measured by how well it does what you said. a coworker is measured by whether the
thing got done. everything above is what it takes to be measured the second way.
